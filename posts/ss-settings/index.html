How to use shadowsocks on Linux

1. Install shadowsocks client

sudo pip install shadowsocks

2. Fix bug in shadowsocks

You can skip this step first, and come back if sslocal does not work. Typically, this happens on the machines with ARM architecture.

Replace string “libcrypto.EVP_CIPHER_CTX_cleanup” to “libcrypto.EVP_CIPHER_CTX_reset” in openssl.py, which is located somewhere like /usr/local/lib/python2.7/dist-packages/shadowsocks/crypto/openssl.py.

3. Write a config file for shadowsocks

I wrote this file in ~/.config, named shadowsocks.json. It contains information like this:

{
    "server": "www.example.com",
    "server_port": 88888,
    "local_address": "127.0.0.1",
    "local_port": 1080,
    "password": "secret",
    "timeout": 600,
    "method": "aes-256-cfb",
    "fast_open": false,
    "workers": 1
}

4. Install Polipo for using shadowsocks in http(s) protocal

sudo apt-get install polipo

5. Modify polipo’s config file

Modify file /etc/polipo/config to this:

logSyslog = false
logFile = /var/log/polipo/polipo.log

socksParentProxy = "127.0.0.1:1080"
socksProxyType = socks5

chunkHighMark = 50331648
objectHighMark = 16384

serverMaxSlots = 64
serverSlots = 16
serverSlots1 = 32

proxyAddress = "0.0.0.0"
proxyPort = 8123

Restart polipo with /etc/init.d/polipo restart after modification.

6. Export environment variable

export http_proxy=127.0.0.1:8123
export https_proxy=$http_proxy
I wrote two alias commands in .bash_aliases for convenience:

alias proxyon="export http_proxy=127.0.0.1:8123 && export https_proxy=127.0.0.1:8123"
alias proxyoff="export http_proxy='' && export https_proxy=''"

7. Usage

  • Invoke shadowsocks: sslocal -c ~/.config/shadowsocks.json;
  • Turn on http_proxy and https_proxy with proxyon;
  • Enjoy;
  • Turn off http_proxy and https_proxy with proxyoff;
  • Turn off shadowsocks with CTRL-C.

Note: This does not apply to git clone. If you want to use shadowsocks in this command, invoke it this way: git clone -c https.proxy=127.0.0.1:8123.

This does not mean you can browse the forbidden websites directly, either. Opening chromium in a terminal with proxyon may solve this problem.
